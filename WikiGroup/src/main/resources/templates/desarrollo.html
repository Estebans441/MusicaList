Versión 1 - Integración (Thymeleaf, Persistencia JPA y JUNIT)

En esta primera versión del despliegue del proyecto, el enfoque principal es establecer la base del sistema mediante el uso de tecnologías como Thymeleaf, JPA y JUNIT.

  1. Frontend con Thymeleaf:

    • Implementar un formulario de contacto en la interfaz de usuario utilizando Thymeleaf como motor de plantillas.

    • Llevar a cabo la estructura y estilo de la página de contacto para asegurar una experiencia de usuario agradable.

  2. Backend con Persistencia JPA:

    • Desarrollar la lógica del backend utilizando Java y JPA para gestionar la persistencia de datos.

    • Crear las entidades necesarias, como "Canción", "Género Musical" y "Usuario", junto con sus relaciones.

    • Implementar las operaciones CRUD (Crear, Leer, Actualizar, Eliminar) para las entidades mencionadas.

  3. Pruebas Unitarias con JUNIT:

    • Escribir pruebas unitarias exhaustivas para las funcionalidades del backend.

    • Asegurarse de que las pruebas cubran escenarios diferentes y bordes para garantizar la robustez del código.

  4. Wiki y Casos de Prueba:

    • Documentar todas las funcionalidades implementadas en la wiki del proyecto.

    • Detallar los casos de prueba que cubren tanto el frontend como el backend, incluyendo escenarios positivos y negativos.





Versión 2 - Integración Angular y Rest

En esta segunda versión, se expande la aplicación incorporando tecnologías modernas como Angular para la interfaz de usuario y se continúa fortaleciendo el backend con REST.

  1. Frontend con Angular:

    • Implementar el mismo formulario de contacto utilizando Angular para crear una interfaz más interactiva y dinámica.

    • Configurar rutas y componentes necesarios para la navegación y presentación de la información.

  2. Backend con REST:

    • Exponer endpoints REST para las operaciones CRUD en el backend.

    • Asegurar que la comunicación entre el frontend y el backend se realice de manera eficiente y segura.

  3. Pruebas Unitarias Continuas:

    • Mantener y mejorar las pruebas unitarias para el nuevo código desarrollado en esta versión.

  4. Wiki y Casos de Prueba Actualizados:

    • Agregar nuevos casos de prueba que cubran las nuevas funcionalidades implementadas en el frontend y el backend.





Versión 3 - Integración (Comunicación, pruebas y seguridad)

En esta tercera y última versión, se abordan aspectos de comunicación, seguridad y pruebas adicionales.

  1. Comunicación con JWT:

    • Implementar la autenticación y autorización utilizando JSON Web Tokens (JWT) para garantizar la seguridad de las interacciones entre usuarios y el sistema.

  2. Historias de Usuario Completas:

    • Desarrollar completamente las historias de usuario relacionadas con el sistema de votación y las funcionalidades de cuenta de usuario..

  3. Pruebas de Interfaz y Pruebas de Carga:

    • Realizar pruebas exhaustivas de la interfaz de usuario para validar que todas las funcionalidades se comporten según lo esperado.

    • Ejecutar pruebas de carga simulando diferentes niveles de tráfico para evaluar el rendimiento y la escalabilidad del sistema.

  4. Documentación y Finalización:

    • Actualizar la documentación en la wiki para incluir detalles sobre la implementación de JWT, las pruebas realizadas y los resultados obtenidos.

Con cada versión, se sigue un proceso de desarrollo iterativo y colaborativo, donde los desarrolladores y equipos involucrados trabajan juntos para lograr los objetivos establecidos y mejorar la calidad y funcionalidad del sistema en cada etapa. La documentación y la comunicación efectiva son elementos clave en todo el proceso para garantizar un despliegue exitoso del proyecto.
